# CLAUDE.md — 오목 프로젝트 작업 규칙

## 1. 스펙이 진실의 원천

- 모든 구현은 `docs/` 스펙 문서를 근거로 한다.
- 스펙에 없는 기능을 임의로 추가하지 않는다.
- 판단이 애매하면 구현 전에 반드시 사용자에게 먼저 확인한다.
- **규칙/요구사항 변경은 코드보다 `docs/` 문서를 반드시 먼저 수정한다. 이것은 절대 원칙이다.**
- 스펙 문서 수정 없이 코드를 먼저 작성하는 것은 금지된다.

## 2. 레이어 분리 원칙

```
src/engine/   — 순수 JS 게임 로직 (React/DOM 의존 금지)
src/ai/       — AI 로직 (React/DOM 의존 금지)
src/ui/       — React 컴포넌트 (engine/ai를 호출만 함)
```

- `engine/`과 `ai/`는 어떤 프레임워크에도 의존하지 않는 순수 JS로 유지한다.
- UI에서 게임 로직을 직접 구현하지 않는다. 반드시 engine/ai를 통해 호출한다.

## 3. 테스트 우선 원칙

- 각 Phase 구현 전, 스펙의 완료 기준(Acceptance)을 테스트 파일로 먼저 작성한다.
- 테스트가 없는 engine/ai 코드는 작성하지 않는다.
- Phase 2 금수 로직은 특히 버그가 잦다 — 단순 카운트 금지, 반드시 패턴 매칭 기반으로 구현.

## 4. Phase 진행 규칙

- 현재 Phase의 완료 기준 테스트가 전부 통과(green)해야 다음 Phase로 넘어간다.
- 각 Phase는 이전 Phase 코드를 재사용하고 그 위에 규칙을 얹는다. 재작성하지 않는다.
