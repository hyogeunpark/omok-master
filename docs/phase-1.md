# Phase 1 — 자유 오목 (Free Gomoku)

> 전제: `index.md` §4 공통 규칙, §5 비기능 요구사항. 본 문서는 그 위에 Phase 1 범위를 정의한다.
> 상태: ✅ 프로토타입 완료 (`omok-phase1.jsx`)

---

## 1. 목표

CPU와 끝까지 한 판을 둘 수 있는, 규칙이 가장 단순한 기본 게임을 완성한다. 이후 단계가 얹힐 **뼈대(ENGINE/AI/UI 분리, 히스토리 스택)**를 마련하는 것이 핵심.

## 2. 이 Phase의 규칙

- 흑·백 **동일 규칙**, 금수 없음.
- **5목 이상**이면 승리 — 6목(장목)도 승리로 인정.
- 스왑 없음. 흑이 항상 선공.

## 3. 기능 요구사항

| ID | 기능 | 설명 |
|----|------|------|
| FR-1 | 착수 | 빈 교점 탭으로 돌 놓기. 점유/비차례/종료 시 무시. |
| FR-2 | 승리·무승부 판정 | 5목 이상 승리 / 보드 만석 무승부. |
| FR-3 | CPU AI | 휴리스틱 수 선택 + 난이도 3단계(쉬움/보통/어려움). |
| FR-4 | 되돌리기(undo) | **내 수 + CPU 응수**를 함께 무르고 내 차례로 복귀. 진 뒤에도 가능. |
| FR-5 | 색 선택 | 흑(선공)/백(후공) 선택, 변경 시 새 판. |
| FR-8 | 표시 | 마지막 수 마커, 승리한 5목 강조. |

## 4. 완료 기준 (Acceptance)

- [ ] 사람이 CPU와 한 판을 끝까지 두고 **승/패/무가 정확히** 판정된다.
- [ ] 모든 방향(가로·세로·두 대각)의 5목이 인정된다.
- [ ] 되돌리기가 내 수와 CPU 응수를 함께 무르고, 무른 직후 CPU가 자동으로 다시 두지 않는다.
- [ ] 진 뒤에도 되돌리기가 동작한다.
- [ ] 색 선택(흑/백)에 따라 선공/후공이 올바르게 동작한다(백 선택 시 CPU가 흑으로 선공).
- [ ] 모바일 세로 화면에서 보드가 정사각으로 보이고 탭 착수가 정확하다.

## 5. 선행/의존

- 없음(첫 단계). 단, ENGINE·AI·UI 레이어 분리와 히스토리 스택 구조를 **반드시** 갖춰 다음 Phase가 얹히게 한다(`architecture.md` §2, §3).
