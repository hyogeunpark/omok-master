# 바텀 네비게이션 스펙

> `docs/index.md` §5 비기능 요구사항(반응형) 전제.
> 시작 화면 하단의 4-탭 네비게이션과 각 탭 화면의 구조 및 데이터 스펙을 정의한다.

---

## 1. 개요

시작 화면(StartScreen) 하단에 고정된 네비게이션 바로, 4개 탭을 전환한다.
탭 전환은 화면(페이지) 교체 방식으로 동작하며, 게임 진행 중에는 표시되지 않는다.

---

## 2. 탭 구성

| 순서 | 아이콘 | 라벨 | 화면 |
|------|--------|------|------|
| 1 | ⬡ | 플레이 | `StartScreen` (기존) |
| 2 | ◈ | 규칙 | `RulesScreen` (신규) |
| 3 | ◫ | 기보 | `RecordsScreen` (신규) |
| 4 | ◉ | 프로필 | `ProfileScreen` (신규) |

- 활성 탭: 아이콘·라벨 `--accent` 색상 강조.
- 기본(비활성): `--on-surface-dim` 색상.

---

## 3. 탭 전환 동작

- `App.jsx`에서 `activeTab` 상태를 관리 (`'play' | 'rules' | 'records' | 'profile'`).
- 게임 화면(`Game`)이 렌더링 중일 때는 바텀 네비 전체 미표시.
- 탭 전환 시 스크롤 위치는 초기화된다.

---

## 4. 규칙 탭 (`RulesScreen`)

### 4-1. 화면 구성

정적 설명 화면. 다음 3개 섹션을 세로로 나열한다.

| 섹션 | 내용 |
|------|------|
| 기본 규칙 | 15×15 보드, 교점 착수, 5목 연속 승리, 흑 선공, 보드 만석 무승부 |
| 렌주 금수 | 흑만 적용. 3-3 금수, 4-4 금수, 장목(6목 이상) 금수. 각 항목에 간단한 시각 예시 포함. |
| 타라구치-10 오프닝 | 3단계 절차(1수·2수·3수 배치 → 스왑 선택 → 후보 제시/선택) 요약. |

### 4-2. 구현

- `src/ui/RulesScreen.jsx` (신규)
- 순수 정적 JSX. 외부 데이터 의존 없음.

---

## 5. 기보 탭 (`RecordsScreen`)

### 5-1. 데이터 스키마

`localStorage` 키: `omok_records`
저장 형식: JSON 배열 (최신순 정렬, 최대 100건 — 초과 시 오래된 것부터 삭제).

```ts
type Record = {
  id: string;          // crypto.randomUUID() 또는 Date.now().toString()
  date: string;        // ISO 8601 (e.g. "2026-06-11T07:39:21.000Z")
  difficulty: 'easy' | 'normal' | 'hard';
  myColor: 'B' | 'W';
  result: 'win' | 'lose' | 'draw';
  moves: { row: number; col: number; color: 'B' | 'W' }[];
};
```

### 5-2. 기록 저장 시점

`Game.jsx`에서 게임 종료(승·패·무) 시 `saveRecord(record)` 호출.

### 5-3. 목록 화면

- 기록 없을 때: "아직 대국 기록이 없습니다" 빈 상태 문구 표시.
- 기록 있을 때: 카드 목록 (최신순).
  - 카드 1건 표시 항목: 결과(승/패/무) · 난이도 · 내 색 · 날짜 · 총 수(moves.length).
  - 승: `--accent` 강조, 패: `--on-surface-dim`, 무: 기본.
- 기록 전체 삭제 버튼 (확인 dialog 없이 즉시 삭제).

### 5-4. 기보 상세 (Phase 외 확장 — 현재 미구현)

카드 탭 시 기보 재생 화면은 이 스펙의 범위 밖. 현재는 탭 액션 없음(클릭 무반응).

### 5-5. 구현 파일

- `src/engine/records.js` (신규): `saveRecord`, `loadRecords`, `clearRecords`
- `src/ui/RecordsScreen.jsx` (신규)

## 6. 프로필 탭 (`ProfileScreen`)

### 6-1. 데이터 소스

`records.js`의 `loadRecords()`를 읽어 런타임에 집계. 별도 저장소 없음.

### 6-2. 표시 항목

#### 종합 통계

| 항목 | 표시 |
|------|------|
| 총 대국 수 | 숫자 |
| 승 / 패 / 무 | 각각 숫자 |
| 전체 승률 | `승 / (승+패)` × 100 (소수점 1자리, 무승부 제외) |

#### 난이도별 승률

| 난이도 | 승 / 총 대국 | 승률 |
|--------|-------------|------|
| 쉬움 | n / N | x% |
| 보통 | n / N | x% |
| 어려움 | n / N | x% |

- 대국 기록이 0건이면 통계 항목 대신 "아직 대국 기록이 없습니다" 표시.

### 6-3. 구현 파일

- `src/ui/ProfileScreen.jsx` (신규)

## 8. 완료 기준 (Acceptance)

| ID | 조건 |
|----|------|
| AC-N1 | 바텀 네비 4개 탭 탭 전환이 동작한다 |
| AC-N2 | 게임 화면 진입 시 바텀 네비가 사라진다 |
| AC-N3 | 규칙 탭에서 기본 규칙·금수·오프닝 3섹션이 표시된다 |
| AC-N4 | 게임 종료 후 기보 탭에 해당 기록이 추가된다 |
| AC-N5 | 프로필 탭에서 승/패/무 집계와 난이도별 승률이 표시된다 |
| AC-N6 | 기록 삭제 후 기보·프로필 탭 모두 빈 상태로 전환된다 |
