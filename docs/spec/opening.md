# 타라구치-10 오프닝 (Taraguchi-10 Opening)

> 구현: `src/engine/opening.js`
> 전제: 렌주 금수(`forbidden.md`) 적용 상태에서 진행.

---

## 1. 목적

흑의 선공 이점을 줄이기 위해, 게임 시작 6수까지 정해진 오프닝 절차를 도입한다.
"확장 사각형 + 매 수 직후 스왑" 구조이며, 색은 절차 종료 전까지 잠정적이다.

## 2. 오프닝 절차 (6수)

1. **흑 1수**: 판의 정중앙. → 상대(백)가 Swap 가능.
2. **백 2수**: 중앙 3×3 안. → 상대(흑)가 Swap 가능.
3. **흑 3수**: 중앙 5×5 안. → 상대(백)가 Swap 가능.
4. **백 4수**: 중앙 7×7 안.
5. **4수 직후, 백이 두 가지 중 하나 선택**:
   - **선택 1**: 상대(흑)가 Swap 가능 → 흑 5수를 중앙 9×9 안에 둠 → 상대(백)가 Swap 가능 → 백 6수를 임의 위치.
   - **선택 2**: 흑이 5수 **후보 10개**를 임의 위치에 제시(대칭 중복 불가) → 백이 그중 하나를 골라 5수 확정 → 백 6수를 임의 위치.
6. **6수 이후** 일반 렌주(공통 규칙 + 흑 금수)로 진행.

## 3. 스왑 권리자 (단계별)

| 단계 | 권리자 |
|------|--------|
| 1수 후 | 백 |
| 2수 후 | 흑 |
| 3수 후 | 백 |
| 선택 1 — 5수 전 | 흑 |
| 선택 1 — 5수 후 | 백 |
| 선택 2 | 스왑 없음 |

## 4. 기능 요구사항

| ID | 기능 | 설명 |
|----|------|------|
| FR-7 | 오프닝 상태기계 | §2 절차를 단계별로 진행(허용 영역 제약·단계 전이). |
| FR-7a | 스왑 UI | 각 swap 시점에서 스왑 권리자에게 "Swap" 버튼 표시. 버튼을 누르지 않고 돌을 두면 교환 없이 진행. 자세한 UX는 `swap.md` 참고. |
| FR-7b | 5수 후보 제시/선택 UI | 선택 2에서 후보 10개 입력(대칭 중복 금지) 및 1개 선택 흐름. |
| FR-7c | CPU 스왑 판단 | CPU가 포석을 평가해 색 교환/후보 선택을 결정. |

## 5. 구현 주의

- **잠정 색 ↔ 확정 색** 전환 시 흑 금수 적용 시점(6수 이후)이 어긋나지 않도록 주의.
- **선택 1/2 결정권자**는 4수를 놓은 **백(White)** (RIF Taraguchi-10 §3 기준).
- **선택 2 후보**: 흑이 10개 제시, 천원 기준 8방향 대칭으로 겹치는 위치 불가. 백이 1개 선택 후 흑의 5수로 확정.
- 오프닝 절차 중에는 금수 판정 없음. 6수 완료 후부터 적용.
