# 타이머 스펙 — Fischer 클록

> 근거: 렌주국제연맹(RIF) 국제 대회 시간 규정.
> 이 문서는 게임 내 시간 제한 규칙, 프리셋, UI 스펙을 정의한다.

---

## 1. 알고리즘 — Fischer 클록

- **방식**: `기본 시간(main)` + `착수당 보너스(increment)` 추가.
- **착수 완료 시**: 방금 착수한 플레이어의 남은 시간에 `increment`를 더한다.
- **시간 초과**: 한 쪽의 시간이 0이 되면 해당 플레이어 패배.
- **오프닝**: 별도 클록 없음. 오프닝 착수도 동일 클록에서 차감.

### 1-1. 클록 진행 규칙

| 상황 | 진행 중인 클록 |
|------|----------------|
| 플레이어 착수 차례 (오프닝 포함) | 플레이어 클록 |
| CPU 연산 중 (`thinking = true`) | CPU 클록 |
| 게임 종료 / 시간 초과 | 중지 |

---

## 2. 시간 프리셋

**고정값: 무제한.** 시작 화면에서 시간 설정 선택 UI를 제공하지 않는다.

| 프리셋 키 | 표시 | 기본 시간 | 착수 보너스 | 비고 |
|-----------|------|-----------|-------------|------|
| `unlimited` | 무제한 | ∞ | 0초 | 클록 미표시 (유일한 설정) |

- 클록 UI는 표시하지 않음.

---

## 3. UI 스펙

### 3-1. 시작 화면

- 시간 설정 섹션 없음. 난이도 카드 클릭 시 바로 게임 시작.
- start-rules 푸터에서 "Fischer 클록" 항목 제거.

### 3-2. 게임 화면 — 클록 표시

- `game-side` 패널 내 `game-clocks` 섹션에 양쪽 클록 표시.
- 나 클록 (왼쪽) / CPU 클록 (오른쪽).
- **활성 클록**: 밝은 색. **비활성 클록**: 흐린 색.
- **30초 미만**: 빨간색(`--error`) + 1초 펄스 애니메이션.
- 형식: `MM:SS` (고정 폭 tabular-nums).

### 3-3. 시간 초과 처리

- ResultOverlay에 "시간 초과" 또는 "시간 초과 승리" 서브 텍스트 표시.
- 내부적으로 `timeoutLoser('player' | 'cpu')` 상태로 관리.
