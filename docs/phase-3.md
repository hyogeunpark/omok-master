# Phase 3 — 타라구치-10 오프닝 (Taraguchi-10 Opening)

> 전제: `index.md` §4 공통 규칙 + Phase 1·2 완료. 본 문서는 게임 시작부의 오프닝(스왑) 절차를 추가한다.
> 상태: 🔄 진행 중

---

## 1. 목표

흑의 선공 이점을 더 줄이기 위해, 게임 시작 6수까지 정해진 오프닝 절차를 도입한다. "확장 사각형 + 매 수 직후 스왑"이 핵심이며, 색은 절차 종료 전까지 잠정적이다.

## 2. 이 Phase의 절차 (오프닝 6수)

1. **흑 1수**: 판의 정중앙. → 상대가 swap(색 교환) 가능.
2. **백 2수**: 중앙 3×3 안. → 상대가 swap 가능.
3. **흑 3수**: 중앙 5×5 안. → 상대가 swap 가능.
4. **백 4수**: 중앙 7×7 안.
5. **4수 직후, 두 진행 중 하나 선택**:
   - **선택 1**: 상대 swap 가능 → 흑 5수를 중앙 9×9 안에 둠 → 상대 swap 가능 → 백 6수를 임의 위치.
   - **선택 2**: 흑이 5수 **후보 10개**를 임의 위치에 제시(대칭으로 겹치는 위치 불가) → 백이 그중 하나를 골라 5수 확정 → 백 6수를 임의 위치.
6. 6수 이후 일반 렌주(공통 규칙 + Phase 2 흑 금수)로 진행.

## 3. 기능 요구사항

| ID | 기능 | 설명 |
|----|------|------|
| FR-7 | 오프닝 상태기계 | §2 절차를 단계별로 진행(허용 영역 제약·단계 전이). |
| FR-7a | 스왑 UI | 각 swap 시점에서 사람은 색 교환 여부 선택. |
| FR-7b | 5수 후보 제시/선택 UI | 선택 2에서 후보 10개 입력 및 1개 선택 흐름. |
| FR-7c | CPU 스왑 판단 | CPU가 포석을 평가해 색 교환/후보 선택을 결정. |

## 4. 완료 기준 (Acceptance)

- [ ] 1~4수의 **확장 사각형(중앙→3×3→5×5→7×7)** 제약이 강제된다.
- [ ] 각 swap 시점에서 색 교환이 올바르게 처리되고 잠정 색이 정확히 반영된다.
- [ ] 선택 1/선택 2 분기가 동작하고, 선택 2의 후보 10개 제시·대칭 중복 금지·1개 선택이 동작한다.
- [ ] CPU가 각 swap 시점에서 합리적으로 색을 선택한다(명백히 불리한 색을 떠안지 않음).
- [ ] 오프닝(6수) 종료 후 일반 렌주(흑 금수 포함)로 매끄럽게 이어진다.
- [ ] Phase 1·2 기능에 회귀가 없다.

## 5. 선행/의존

- Phase 2 완료. 오프닝은 `src/engine/opening.js`(상태기계)에 신설.
- CPU 스왑 판단은 Phase 1의 위치 평가 함수(`src/ai/evaluate.js`)를 **"포석 우열 판단"**에 재사용.

## 5-1. RIF 규정 확정 사항

- **선택 1/2 결정권**: 4수를 놓은 **백(White)**이 결정. (RIF Taraguchi-10 §3 기준)
- **스왑 권리 보유자** (각 단계별):
  - 1수 후: 백이 스왑 가능
  - 2수 후: 흑이 스왑 가능
  - 3수 후: 백이 스왑 가능
  - 선택 1에서 5수 전: 흑이 스왑 가능
  - 선택 1에서 5수 후: 백이 스왑 가능
- **선택 2 후보**: 흑이 10개 제시, 대칭 중복 금지(천원 기준 8방향 대칭). 백이 1개 선택 후 흑의 5수로 확정. 이후 백이 6수를 임의 위치에 착수. 스왑 없음.
- **금수 적용 시점**: 6수 완료 후부터. 오프닝 절차 중에는 금수 판정 없음.

## 6. 구현 주의

- **선택 1/선택 2의 결정권자**와 **각 단계 swap 권리 보유자**의 정확한 정의는 **RIF 공식 규정**을 대조해 확정할 것. 본 문서는 절차의 골격을 정의한다.
- 잠정 색 ↔ 확정 색 전환 시 흑 금수 적용 시점(6수 이후)이 어긋나지 않도록 주의.

## 7. 이 Phase 범위 밖

- 고모쿠식 자유 스왑(Swap/Swap2)은 채택하지 않음(렌주 계열인 타라구치-10만 적용).
