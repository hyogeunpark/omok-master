# 렌주 금수 규칙 (Renju Forbidden Moves)

> 흑에게만 적용. 백은 금수 없음.
> 구현: `src/engine/forbidden.js`

---

## 1. 금수 종류

- **삼삼 (3-3)**: 한 수로 "열린 3"을 동시에 2개 이상 만드는 수 → 금수.
- **사사 (4-4)**: 한 수로 "4"를 동시에 2개 이상 만드는 수 → 금수.
- **장목 (6목 이상)**: 흑은 **정확히 5목**으로만 승리. 6목 이상은 승리 아님(금수).
- 백은 금수 없음. 백은 장목으로도 승리.

## 2. 판정 우선순위

1. 어떤 수가 **정확히 5목**을 완성하면 → 흑 승리. (같은 수가 형식상 3-3/4-4여도 5목이 우선)
2. 흑이 **6목 이상**을 만드는 수 → 금수(승리 아님).
3. 5목이 아니면서 **3-3 또는 4-4** → 금수.
4. 금수 자리에 흑이 착수하면 흑 **반칙패**.

## 3. UI 요구사항

| ID | 기능 | 설명 |
|----|------|------|
| FR-6 | 금수 판정·표시 | 흑 차례에 금수 자리를 사전 시각 표시하고, 착수 시 §2 우선순위로 판정. |
| FR-2′ | 승리 판정 수정 | 흑은 정확히 5목 승리, 백은 5목 이상 승리로 분기. |

## 4. 구현 주의

- "열린 3", "4", 가짜(거짓) 금수, 끊긴 패턴(`O_OO`, `OO_O`), 막힌 끝을 정확히 분석해야 한다. **단순 연속 카운트로는 부정확** → 윈도우/패턴 매칭 기반으로 구현.
- "거짓 금수"(상대 돌이 닿아 실제로는 금수가 아닌 경우) 등 예외는 **RIF 공식 규정**을 대조해 점진 보강.
