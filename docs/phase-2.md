# Phase 2 — 렌주 금수 (Renju Forbidden Moves)

> 전제: `index.md` §4 공통 규칙 + Phase 1 완료. 본 문서는 흑 금수 규칙을 추가한다.
> 상태: ✅ 완료

---

## 1. 목표

선공(흑)의 이점을 제한하는 렌주 금수를 도입한다. 흑에게만 적용하며, Phase 1의 5목 판정 위에 **예외와 우선순위**를 얹는다.

## 2. 이 Phase의 규칙 (흑에게만 적용)

- **삼삼 (3-3)**: 한 수로 "열린 3"을 동시에 2개 이상 만드는 수 → 금수.
- **사사 (4-4)**: 한 수로 "4"를 동시에 2개 이상 만드는 수 → 금수.
- **장목 (6목 이상)**: 흑은 **정확히 5목**으로만 승리. 6목 이상은 승리 아님(금수).
- 백은 금수 없음. 백은 장목으로도 승리.

### 판정 우선순위 (구현 시 반드시 준수)

1. 어떤 수가 **정확히 5목**을 완성하면 → 흑 승리. (같은 수가 형식상 3-3/4-4여도 5목이 우선)
2. 흑이 **6목 이상**을 만드는 수 → 금수(승리 아님).
3. 5목이 아니면서 **3-3 또는 4-4** → 금수.
4. 금수 자리에 흑이 착수하면 흑 **반칙패**.

## 3. 기능 요구사항

| ID | 기능 | 설명 |
|----|------|------|
| FR-6 | 금수 판정·표시 | 흑 차례에 금수 자리를 사전 시각 표시하고, 착수 시 §2 우선순위로 판정. |
| FR-2′ | 승리 판정 수정 | 흑은 정확히 5목 승리, 백은 5목 이상 승리로 분기. |

## 4. 완료 기준 (Acceptance)

- [ ] 흑이 3-3, 4-4, 장목 자리를 둘 수 없다(또는 두면 반칙패 처리).
- [ ] **5목 우선** 판정이 정확하다(5목이면 3-3/4-4여도 승리).
- [ ] 백은 장목(6목 이상)으로도 승리한다.
- [ ] 흑 차례의 금수 표시가 실제 판정과 100% 일치한다.
- [ ] CPU가 자신이 흑일 때 금수 자리를 두지 않는다.
- [ ] Phase 1의 기능(undo·색선택·표시)이 금수 도입 후에도 정상 동작(회귀 없음).

## 5. 선행/의존

- Phase 1 완료(ENGINE 분리 필수). 금수 판정은 `src/engine/forbidden.js`에 신설하고 ENGINE에서만 호출.

## 6. 구현 주의

- "열린 3", "4", 가짜(거짓) 금수, 끊긴 패턴(`O_OO`, `OO_O`), 막힌 끝을 정확히 분석해야 한다. **단순 연속 카운트로는 부정확** → 윈도우/패턴 매칭 기반으로 구현.
- "거짓 금수"(상대 돌이 닿아 실제로는 금수가 아닌 경우) 등 예외는 **RIF 공식 규정**을 대조해 점진 보강. 이 부분에서 버그가 가장 잦으므로 테스트 케이스를 먼저 작성할 것.
